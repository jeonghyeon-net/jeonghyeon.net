# CSS 없이 만드는 이유

브라우저의 기본 스타일은 충분히 읽을 만하다. serif 폰트, 적당한 여백, 링크의 파란색 밑줄. 1990년대부터 작동해온 것들이다.

CSS를 추가하는 순간 책임이 생긴다. responsive 대응, dark mode, 브라우저 호환성, 접근성 점검. 글을 쓰려고 만든 사이트에서 스타일 작업에 더 많은 시간을 쓰게 된다.

## 기본 스타일의 장점

- 모든 브라우저에서 동작한다
- 로딩 속도가 가장 빠르다
- 유지보수할 코드가 없다
- 접근성이 기본으로 보장된다

## 단점

못생겼다. 하지만 내용이 좋으면 상관없다고 생각한다.
